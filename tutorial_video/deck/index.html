SECTION FOUR
.gitignore 的順序陷阱
防呆的第一道關卡：.gitignore
一個隱藏檔。第一行寫上 data/
整個資料夾，Git 就當作不存在。
致命陷阱：它不會回溯
  • .gitignore 只對「還沒被追蹤」的檔案有效。
  • 先 commit、才補 .gitignore：data/ 已經在追蹤清單，照樣會上傳。
  • 順序是關鍵：先寫好 .gitignore，再做第一次 commit。
一句話：先關門，再搬東西。
已經不小心追蹤了？可以這樣移除追蹤、保留本機檔案：
git
$ git rm -r --cached data/
# 從 Git 追蹤移除，但本機 data/ 還在

出生年月日→年齡

把可識別病人資料交給雲端 AI＝把個資傳出院外，明確違法。
對照表留在院內系統，永遠不放進專案資料夾。

SECTION SEVEN
資料夾不是圍牆，機器才是
你可能想：個資放資料夾外，
AI 只在資料夾裡，應該安全了吧？
很遺憾，這個假設不成立。
AI 的「工作資料夾」是預設範圍，不是一道圍牆
  • 它的寫入被限制在專案內，但讀取不是——它用的是你的權限。
  • 你以為 AI 只待在一個房間，其實它拿著整棟房子的鑰匙。
  • 它開別的房間會敲門問你，但用久了，大家都會按同意按到麻痺。
最可靠的不是「把門關好」，
而是讓房子裡根本沒有貴重物品。
這在資安叫「降低攻擊面」
個資　和　雲端 AI，
永遠不要在同一台機器。

⚠️

Dangerously skip permissions

啟動 AI 工具時，你會看到這個選項。
意思是：AI 之後任何操作都不再問你，直接做。

建議：初學者一律不要開。
權限詢問，是你最後一道防線。

方便和安全，是用環境換的，
不是用運氣換的。
