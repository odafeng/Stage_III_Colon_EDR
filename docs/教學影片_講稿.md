*（畫面：五個編號步驟）*
> 接下來，我會帶你用 AI，從零開始完成一篇研究的分析。整個流程，只有五個步驟。

### 0:15 ｜STEP 1
*（畫面：大大的「01 建立結構化資料夾」）*
> 第一步，先把你的研究，整理成結構化的資料夾。

*（畫面：資料夾樹狀展開）*
> 分成四個固定的資料夾：Data 放資料、Figures 放圖、Tables 放表格，而最重要的 Notebooks，放你所有的分析程式。

### 0:27 ｜終端機建資料夾
*（畫面：終端機打字 mkdir）*
> 在終端機裡，一行 mkdir 指令，就能把這四個資料夾一次建好，不用一個一個用滑鼠點。

### 0:34 ｜STEP 2
*（畫面：「02 GitHub ＋ Git 版本控制」）*
> 第二步，建立一個 GitHub Repo，用 Git 做版本控制。

### 0:38 ｜Git 指令
*（畫面：git init / add / commit / push 與 repo 卡片）*
> 你只要記得四個動作：init、add、commit、push。每一次 commit，就像遊戲的存檔點——哪天改壞了、電腦壞了，都能救回來。

### 0:49 ｜STEP 3
*（畫面：「03 與 AI Agent 溝通需求」）*
> 第三步，也是最關鍵的一步：怎麼跟 AI 溝通。

### 0:53 ｜對話示範
*（畫面：你 ↔ AI 對話泡泡）*
> 重點有兩個。第一，清楚說出你的「預期成果」和「分析方法」，例如：用 XGBoost 加交叉驗證，預測十八個月內的早期遠端復發。第二，一定要請 AI 把整個分析過程，寫成一個一個的 Notebook 檔案。

### 1:05 ｜為什麼用 Notebook
*（畫面：可重現性／方便修正 兩張卡）*
> 為什麼一定要寫成 Notebook？因為它有兩個關鍵好處。第一是可重現：Reviewer 要你修改，你只要改那一格、重跑，結果就一致了。第二是方便修正。

### 1:15 ｜可重現性示範
*（畫面：改一行程式 → 圖更新）*
> 看這個例子，我只改了畫圖程式裡的一行顏色，重新執行，右邊的圖就自動更新了。改一個地方、結果立刻重現——這就是把分析寫成程式的威力。

### 1:25 ｜真實圖表

> 這些圖，全部都是用示範資料、由 Notebook 真實跑出來的，每一張的程式碼都留在裡面，隨時可以重現。

### 1:37 ｜實際操作（JupyterLab）
*（畫面：真實 Lab，捲動展示程式碼與輸出）*
> 左邊，就是我們剛剛建立的四個資料夾。打開第二個 Notebook——模型開發。你會看到，每一格程式碼下面，都直接接著它跑出來的結果：資料表、ROC 曲線、校準曲線，還有最後的績效表格。程式、結果、說明，全部留在同一個檔案裡。這，就是可重現研究的核心。

### 1:58 ｜STEP 4
*（畫面：「04 撰寫 Manuscript ＆ 文獻」）*
> 第四步，撰寫論文、尋找文獻。

### 2:02 ｜換你發揮
*（畫面：「分析做完，換你發揮創意」）*
> 分析做完，就換你發揮創意。你可以請 AI 幫忙整理結果、草擬段落，但你，才是研究真正的主導者。

### 2:09 ｜⚠️ 文獻幻覺
*（畫面：紅框警告）*
> 但這裡要特別、特別提醒：文獻幻覺。AI 很會幫你分析、寫程式，但在引用文獻時，它可能編出看起來很真、其實根本不存在的資料。所以，每一篇參考文獻，你都一定要親自查證。

### 2:20 ｜回顧
*（畫面：五項打勾清單）*

### 2:29 ｜片尾

## 🎬 進階：若你要自己錄「實際操作」片段（取代 LIVE DEMO）

下面是給你**邊操作邊念**的延伸講稿，對應你自己錄的 Colab／Claude Code／GitHub 畫面。

**① 建立資料夾（終端機 / Colab）**
> 我們先建立資料夾。我在終端機打 `mkdir -p Data Figures Notebooks Tables`，按下 Enter——四個資料夾一次就建好了。`ls` 確認一下，乾淨俐落。

**② GitHub + Git**
> 接著做版本控制。`git init` 初始化、`git add` 把檔案加進來、`git commit` 存一個版本、最後 `git push` 上傳到 GitHub。你看，Repo 裡就有完整紀錄了，每一筆都可以回溯。

**③ 跟 AI Agent 溝通（Claude Code / ChatGPT）**
> 重點來了，我怎麼跟 AI 講。我會這樣說：我有 Stage three 大腸癌的資料，想預測十八個月的早期遠端復發，請用 XGBoost 加交叉驗證，做特徵選擇和外部驗證——而且，請把每個階段都寫成獨立的 Notebook。你看，它就一步步把 `.ipynb` 建出來了。

**④ 跑 Notebook 看結果**
> 我們打開 Notebook，從上往下一格一格執行。載入資料、訓練模型、畫出 ROC 曲線……每一格的結果，就直接顯示在程式碼下面。要改？改那一格、重跑，整份結果就重現了。

> ⚠️ 錄影前提醒：畫面中不要出現 API Key、信箱、病人資料或私人分頁；資料一律使用 repo 裡的虛構 `synthetic_*.csv`。
